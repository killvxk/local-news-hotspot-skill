# 本地热点新闻分析器

从 worldmonitor 项目验证的权威数据源采集实时新闻，智能评分排名，展示全球热点和本地相关新闻。

## 核心工作流程

### 1. 检测用户位置

使用 ipinfo.io API 确定用户地理位置：

```bash
curl -s https://ipinfo.io/json
```

提取字段：`city`, `region`, `country`, `loc`（经纬度）

如果 API 失败，仅显示全球新闻，跳过本地过滤。

### 2. 采集新闻数据

从多层级数据源获取 RSS feeds，优先使用高可信度来源。

**数据源配置**：详见 `references/rss-sources.md`

**核心来源**：
- **Tier 1**（通讯社）：Reuters, AP, BBC, Bloomberg
- **Tier 2**（主要媒体）：CNN, Guardian, NPR, Al Jazeera
- **区域来源**：根据用户位置动态选择（亚洲、欧洲、美洲、中东、非洲、中国）

**获取命令**：

```bash
curl -s -L -A "Mozilla/5.0 (compatible; NewsBot/1.0)" --max-time 10 "<feed-url>"
```

**提取字段**：
- RSS: `<title>`, `<description>`, `<pubDate>`, `<link>`, `<category>`
- Atom: `<title>`, `<summary>`, `<published>`, `<link href="...">`, `<category term="...">`

**性能优化**：
- 并行获取多个 feeds（后台进程）
- 每个 feed 10 秒超时
- 每个 feed 最多处理 50 个项目

### 3. 评分和排名

应用多维度评分系统对新闻排名。

**评分维度**（总分 120 分）：
- **时效性** (0-40 分)：越新鲜得分越高
- **来源可信度** (0-30 分)：Tier 1 > Tier 2 > Tier 3
- **本地相关性** (0-30 分)：城市 > 省份 > 国家 > 邻国
- **话题重要性** (0-20 分)：突发 > 冲突 > 政治 > 经济 > 常规

**详细算法**：参见 `references/scoring-algorithm.md`

**去重检测**：
- 标题相似度 > 70% 视为重复
- 保留得分最高的版本
- 合并来源信息："报道来源：Reuters, BBC, CNN"

**筛选标准**：
- 全球热点：总分 ≥ 60 分，取前 3-5 条
- 本地新闻：本地相关性 > 0 且总分 ≥ 40 分，取前 3-5 条

### 5. 边缘情况处理

**本地新闻不足**：

```
📍 [位置] 本地新闻
   ℹ️  过去 24 小时内没有重大本地新闻。
   改为显示 [区域] 的区域新闻。
```

**API 失败**：
- 位置检测失败：仅显示全球新闻
- RSS feeds 失败：尝试备用来源，显示部分结果
- 所有来源失败：显示错误消息和故障排除建议

**缓存策略**：
- 结果缓存 15 分钟（`/tmp/news-hotspot-cache-[timestamp].json`）
- 避免频繁 API 调用

**错误恢复**：
- 网络问题：指数退避重试（1s, 2s, 4s）
- XML 解析错误：跳过格式错误的 feeds
- 空结果：扩展时间窗口到 48 小时

## 实现要点

**语言支持**：
- 根据国家代码自动选择区域来源
- 中国/台湾：新华社、CCTV
- 日本：NHK、日经
- 韩国：Yonhap、KBS
- 阿拉伯语国家：Al Jazeera Arabic、Al Arabiya

**隐私保护**：
- 仅通过公共 IP 地理定位
- 不存储或传输个人数据
- 所有处理在本地完成

**性能目标**：
- 总执行时间 < 30 秒
- 并行获取 RSS feeds
- 早期筛选减少内存占用

## 成功标准

执行成功应满足：
- ✅ 在 30 秒内完成
- ✅ 返回 3-5 条全球热点（总分 ≥ 60）
- ✅ 返回 3-5 条本地新闻（如果可用，总分 ≥ 40）
- ✅ 清晰可读的终端输出
- ✅ 所有新闻包含有效 URL
- ✅ 准确的时间戳和位置信息
